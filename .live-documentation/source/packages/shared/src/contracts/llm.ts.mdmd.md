## Authored
### Purpose
Specifies the shared request contract the extension uses to ask the server to invoke an LLM and defines the shape of the response payload that returns completions plus usage metadata.

### Notes
- The payload accepts arbitrary `schema` and tagging metadata so server adapters can translate structured prompts without leaking implementation types across the boundary.
- Results surface either raw text or structured objects, and expose the resolved `modelId` and token usage to keep telemetry consistent with the orchestrator.
